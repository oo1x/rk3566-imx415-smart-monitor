# NPU 900 MHz、RKNN profiling 与逐节点延迟补充测试

测试日期：2026-07-16

## 测试条件

- 用户确认电源已为 5V3A；软件没有电压/电流遥测，因此这是测试条件说明，不是测量值。
- IMX415：1920×1080、NV12、V4L2 实际约 60 fps。
- YOLO11n：640×640，S4 `latest_if_idle` 异步策略，每 5 个已处理帧检查一次推理。
- RGA overlay、MPP H.264、RTSP-over-TCP，VM FFmpeg 持续解码。
- 关闭 RKNN profiling 的 600/900 MHz 正式窗口分别约 121 秒；profiling 另开短轮，避免采样开销污染正式结果。
- 为了不把不同画框负载混为频率效果，第一轮无框负载的 600 MHz 数据被保留但排除；正式对比使用“绝大多数帧实际叠框”的 600 MHz 补跑和 900 MHz 轮。

## 频率对 NPU 的真实收益

| 指标 | 600 MHz | 900 MHz | 变化 |
|---|---:|---:|---:|
| 关闭 profiling 的 NPU P50 | 56.076 ms | 49.303 ms | -12.08% |
| 关闭 profiling 的 NPU P95 | 58.878 ms | 52.168 ms | -11.39% |
| SDK profiling `PERF_RUN` | 70.072 ms | 65.314 ms | -6.79% |
| 138 个 NPU 层合计 | 48.818 ms | 42.765 ms | -12.40% |
| 14 个 CPU 层合计 | 12.789 ms | 14.085 ms | +10.13% |

逐层报告显示，注意力模块中的 `exSoftmax13:/model.10/m/m.0/attn/Softmax` 在 CPU 执行，600/900 MHz 都接近 10 ms；另有 CPU Transpose。NPU 拉满不能加速这些算子，所以完整 RKNN 调用收益明显小于 NPU 层自身收益。

profiling 通过 `RKNN_FLAG_COLLECT_PERF_MASK`、`RKNN_QUERY_PERF_RUN` 和 `RKNN_QUERY_PERF_DETAIL` 实现。实验版本只在设置 `RKNN_PROFILE_OUTPUT` 时开启，未覆盖 S4 稳定二进制；profiling 本身有开销，不应作为生产 FPS。

## 同场景全链路 A/B

| 指标 | 600 MHz 补跑 | 900 MHz |
|---|---:|---:|
| V4L2 采集 FPS | 60.000 | 60.008 |
| 处理/RTSP 事件 FPS | 48.084 | 43.825 |
| 推理完成 FPS | 9.617 | 8.763 |
| 最新帧替换率 | 19.86% | 26.97% |
| NPU 平均负载 | 40.12% | 30.70% |
| 进程 CPU | 33.51% | 31.29% |
| SoC 平均/最高 | 60.457/62.222°C | 61.661/63.333°C |
| FFmpeg 错误日志 | 0 字节 | 0 字节 |

900 MHz 确实缩短了 NPU 段，但没有缩短 RGA overlay、MPP 和 RTSP 段。本轮 900 MHz 的 overlay 尾延迟更高，导致完整输出较低；可能涉及共享内存带宽/RGA 竞争及画框复杂度的细微变化。这里只完成一轮配对，不能把差异写成确定因果，后续需要多轮随机化 A/B。

## V4L2 时间戳到各节点

当前每帧 V4L2 flag 均为 `0x00002001`，包含单调时钟标志，但不包含 `V4L2_BUF_FLAG_TSTAMP_SRC_SOE`。因此以下起点是 V4L2 EOF 时间戳，不是 IMX415 曝光开始。

| EOF 到节点 | 600 MHz P50/P95/P99 | 900 MHz P50/P95/P99 |
|---|---:|---:|
| DQBUF | 0.199/0.479/0.735 ms | 0.200/0.307/0.654 ms |
| RGA 预处理结束（推理帧） | 8.554/9.343/9.851 ms | 8.560/9.369/9.746 ms |
| NPU 结束（推理帧） | 64.745/67.863/69.254 ms | 57.900/60.973/62.368 ms |
| YOLO 后处理结束（推理帧） | 67.471/70.713/72.194 ms | 60.560/63.664/65.221 ms |
| Overlay 完成 | 2.500/9.714/10.062 ms | 3.096/10.758/11.671 ms |
| MPP 编码输出 | 15.070/22.372/22.731 ms | 15.682/23.352/24.293 ms |
| `rtsp_tx_video` 返回 | 15.731/23.063/31.557 ms | 16.406/24.360/25.900 ms |
| `rtsp_do_event` 完成 | 15.749/23.084/31.573 ms | 16.417/24.377/25.918 ms |

`rtsp_do_event` 完成表示应用完成本轮 RTP-over-RTSP/TCP socket 发送尝试，不等于最后一个比特离开网卡，也不等于客户端显示。真正的“曝光开始→各节点”还缺少 SOE→EOF 段，需要驱动 SOE 时间戳、IMX415 XVS/GPIO 或外部逻辑分析仪/示波器；该段没有被估算或编造。

## 稳定性边界

900 MHz 正式窗口未触发 85°C 自动保护，最高 63.333°C；profiling 轮同样完成，FFmpeg 错误日志为 0。测试结束恢复 `userspace / 600 MHz`，没有测试进程、8554 监听或 `/dev/video0` 占用。

本次只证明约 2 分钟正式窗口和短 profiling 可运行，不等同于 30 分钟以上的 900 MHz 长时稳定性结论。
